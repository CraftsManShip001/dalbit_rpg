# 회고

## 잘된 점

- 달빛약속 런타임 실행을 Web Worker로 격리해 UI 프리징 리스크를 초기에 제거했다.
- 판정 로직(`useJudge`)과 상태 변경(`BattlePage`, `gameStore`) 책임을 분리해 구조를 유지했다.
- 사용자 피드백을 빠르게 반영해 타이머 가시성, 단축키 실행, 자동 포커스, 레이아웃 고정성을 개선했다.
- 엔드리스 층 시스템과 최고기록 저장으로 반복 플레이 동기를 강화했다.

## 아쉬웠던 점 및 개선 방안

- 초기 문서 일부가 과거 스테이지 모델(`stageclear`) 기준으로 남아 코드와 불일치가 발생했다.
  - 개선: 기능 변경 시 문서 동기화 체크리스트를 릴리즈 단계에 포함
- 문제셋이 대량으로 확장되면서 난이도 곡선 검증이 충분히 자동화되지 못했다.
  - 개선: 문제 타입별 샘플 회귀 테스트(정답/오답/타임아웃) 자동화
- UI 수정 과정에서 레이아웃 회귀(깨짐)가 일시적으로 발생했다.
  - 개선: 주요 해상도 스냅샷 테스트 또는 플레이 회귀 시나리오 고정

## 시간이 더 있었다면 하고 싶었던 것

- `useJudge` 및 `useGameEngine` 단위 테스트/통합 테스트 추가
- 문제 카테고리/난이도 태깅과 층별 큐레이션 개선
- 전투 로그 히스토리(최근 판정/피해/회복) 패널 추가
- 모바일 대응을 위한 전용 HUD/입력 UI 최적화
