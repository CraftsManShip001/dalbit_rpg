# 아이디에이션 — 달빛약속 코딩 RPG

## 출발점

`@dalbit-yaksok/core`를 활용해 "코드를 실행해보는 도구"를 넘어서, 코드 작성 자체가 플레이가 되는 구조를 만들고자 했다.

핵심 질문은 하나였다.

> 달빛약속 문법 연습을 사용자가 스스로 반복하게 만드는 인터랙션은 무엇인가?

## 핵심 아이디어

RPG 전투 루프를 학습 루프로 바꿨다.

- 코드를 제출하면 전투 행동으로 변환
- 정답이면 공격 또는 회복
- 오답이면 패널티를 받아 긴장 유지
- 적은 주기적으로 자동 공격

즉, 학습 동기는 "문법 이해"가 아니라 "살아남아 더 높은 층에 도달"하는 것으로 설계했다.

## 문제 유형 설계

### 빠른 문제 (quick)

- 목적: 출력/연산 기초 반복
- 특성: 짧은 제한시간, 즉시 채점
- 현재 규칙: 정답 시 공격, 오답 시 소량 HP 감소

### 따라치기 (copy)

- 목적: 달빛약속 문법 형태 체화
- 특성: 실행 없이 문자열 일치 판정
- 현재 규칙: 정답 시 회복, 오답 시 소량 HP 감소

### 알고리즘 (algorithm)

- 목적: 반복/조건/목록 기반 문제 해결
- 특성: 긴 제한시간, 고위험 고보상
- 현재 규칙: 정답 시 큰 공격, 오답/타임아웃 시 큰 HP 감소

## 엔드리스 구조로 확장한 이유

초기의 고정 스테이지 구조보다 엔드리스 층 구조가 반복 학습과 몰입에 더 유리했다.

- 몬스터 1마리 처치마다 다음 층
- 층이 오를수록 적 공격과 문제 가중치가 점진적으로 상승
- 최고기록을 `localStorage`에 저장해 재도전 루프 강화

## 달빛약속 특성 반영

- 한글 키워드(`만약`, `반복`, `보여주기`)를 그대로 학습 루프로 노출
- 따라치기에서 4칸 들여쓰기 정확성 체화
- 목록 인덱스(1부터 시작) 등 언어 특성을 문제로 반복 노출

## UX 방향

- 문제 타이머/적 공격 타이머를 분리해 가시화
- `Cmd/Ctrl + Enter` 실행으로 반복 입력 속도 확보
- 문제 전환 시 에디터 자동 포커스로 입력 흐름 유지
- 긴 문제는 내부 스크롤로 처리해 한 화면 레이아웃 유지

## 왜 이 방향인가

교육형 툴은 동기 유지가 가장 큰 리스크다.
전투 루프 기반 피드백(데미지, 회복, 타이머 압박)은 사용자가 자발적으로 코드를 반복 입력하게 만드는 강한 동기 구조다.

결론적으로 이 프로젝트의 목적은 "달빛약속 문법 설명"이 아니라,
"달빛약속 코드 작성을 반복하게 만드는 게임 루프 설계"다.
