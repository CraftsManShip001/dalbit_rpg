## 에이전트에게 위임한 작업의 범위

- `BattlePage`에 전투 메인 레이아웃(HUD → 문제 → 에디터/결과/실행)을 구성
- `useGameEngine`, `useQuestionGen`, `useJudge` 연결
- `idle -> questioning` 진입 시 문제 선택 및 코드 초기화
- 판정 후 `result` 페이즈를 거쳐 자동으로 다음 문제 전환
- 적 처치 시 `advanceFloor()`로 다음 층 진행
- 타이머/팝업 관련 timeout, rAF cleanup 처리

## 위임 판단 근거

레이아웃과 게임 흐름 제어가 동시에 들어가는 파일이라 변경량이 크고 회귀 위험이 높다.
에이전트 구현 후 리뷰와 브라우저 검증을 병행해 안정성을 확보했다.

## 에이전트 산출물 검증 방법

1. 흐름 검증
- 시작 직후 문제 진입
- 제출 후 판정/피드백
- 1.5초 후 다음 문제 자동 전환
- 적 처치 시 층 증가

2. 타이머 검증
- 문제 타이머 만료 시 자동 오답 처리 + 다음 문제 진행
- 시작 직후 즉시 오답 처리되는 레이스 컨디션 교정 확인

3. UX 검증
- 전역 `Cmd/Ctrl + Enter` 실행
- 문제 전환 시 에디터 자동 포커스
- 긴 문제에서도 에디터 영역 최소 높이 유지

4. 정적 검증
- `pnpm build` 통과

## 에이전트가 잘못된 방향을 제시했을 때 교정 방법

- 과거 `stageclear` 기반 분기 설명을 제거하고, 현재 엔드리스 층 진행 흐름으로 문서를 갱신했다.
