# 에이전트 협업 기록 템플릿

> 각 태스크 완료 후 바로 작성할 것. 나중에 몰아서 쓰면 기억이 흐릿해진다.

---

## [태스크 번호 + 이름] 예: T02 — 달빛약속 런타임 POC

### 사용한 에이전트 도구

- [ ] Codex
- [ ] Claude
- [ ] antigravity
- [ ] 직접

### 에이전트에게 위임한 작업의 범위

> 무엇을 시켰고, 어디까지 맡겼는가

예: YaksokSession을 Web Worker에서 실행하는 격리 환경 구축.
Worker 통신 프로토콜 설계부터 타임아웃 처리까지 전부 위임.

### 위임 판단 근거

> 왜 이 작업을 에이전트에게 맡겼는가

예: Worker 보일러플레이트는 패턴이 명확하고 달빛약속 도메인 지식 없이도 구현 가능.
반면 출력 캡처 방식 결정은 달빛약속 API 이해가 필요해서 직접 확인 후 명세를 줬다.

### 에이전트 산출물 검증 방법

> 어떻게 확인했는가

예:
1. Claude 코드 리뷰: Worker terminate 누락 없는지, any 타입 없는지 확인
2. 무한루프 코드 직접 실행 → 5초 후 TIMEOUT 반환 확인
3. antigravity로 UI 블로킹 없음 확인 (Worker 실행 중 버튼 클릭 테스트)

### 에이전트가 잘못된 방향을 제시했을 때 교정 방법

> 있었으면 기록, 없으면 "없음"

예: Codex가 Worker를 재사용하는 방식으로 구현했으나,
무한루프 발생 시 terminate 후 재생성이 불가능해서 매 실행마다 새 Worker 생성 방식으로 재요청.
