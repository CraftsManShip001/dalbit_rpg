# T06 — 문제 데이터

## 사용한 에이전트 도구

- [x] Codex
- [x] Claude
- [ ] antigravity
- [x] 직접

## 에이전트에게 위임한 작업의 범위

- `src/data/questions.ts` 문제 타입/데이터 확장
- `quick`, `copy`, `algorithm` 대량 문제셋 반영
- `src/data/stageConfig.ts` 기본 난이도/적 로테이션 데이터 구성
- 데이터 로직 분리 원칙 유지(순수 데이터)

## 위임 판단 근거

문제 데이터는 반복 입력 작업량이 크고 실수 가능성이 높다.
에이전트로 초안 구성 후, 사람이 문법/정답/시간제한을 검수하는 방식이 가장 효율적이었다.

## 에이전트 산출물 검증 방법

1. 타입 검증
- discriminated union(`quick | copy | algorithm`) 일치 확인

2. 데이터 검증
- 문제 텍스트/정답/제한시간 누락 여부 확인
- 따라치기 코드 들여쓰기(4칸) 점검

3. 런타임 검증
- 샘플 문제를 실제 실행해 `expectedOutputs` 정합 확인

4. 정적 검증
- `pnpm build` 통과

## 에이전트가 잘못된 방향을 제시했을 때 교정 방법

- 초기에 축약 문제셋으로 시작했으나, 이후 사용자 제공 대량 문제셋을 기준으로 확장 반영했다.
- 최종 기준은 현재 `questions.ts`의 확장 데이터다.
