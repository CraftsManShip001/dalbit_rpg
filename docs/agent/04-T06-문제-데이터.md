- [x] Codex
- [x] Claude

## 에이전트에게 위임한 작업의 범위

- Codex에 `src/data/stageConfig.ts`, `src/data/questions.ts` 구현을 위임했다.
- `Question` discriminated union(`quick`/`copy`/`algorithm`) 타입 정의와 스테이지별 문제 데이터 구성을 맡겼다.
- 데이터 완료 기준 충족 여부까지 함께 검증하도록 위임했다.

## 위임 판단 근거

- T06은 서비스 로직 없이 타입/정적 데이터 구성 비중이 높아 Codex 위임 효율이 높았다.
- 다만 알고리즘 정답 출력값은 판정 정확도에 직접 영향이 있으므로, 런타임 직접 실행 검증을 필수 조건으로 두고 진행했다.
- 최종적으로 타입 안정성과 태스크 체크리스트 충족 여부를 Claude 리뷰로 교차 확인했다.

## 에이전트 산출물 검증 방법

1. 데이터 구성 검증:
   stage 1은 `quick 4개 / copy 3개 / algorithm 3개`, stage 2는 각 유형 3개로 구성됐는지 확인했다.
2. 가중치 검증:
   `questionWeights` 합계를 stage 1, stage 2 각각 계산해 모두 `1.00`인지 확인했다.
3. 따라치기 코드 형식 검증:
   `codeToType` 들여쓰기가 탭 없이 4칸 스페이스인지 확인했다.
4. 알고리즘 정답 검증:
   `@dalbit-yaksok/core` 런타임으로 알고리즘 문제 답안 코드를 직접 실행해 `expectedOutputs` 일치 여부를 확인했다.
5. 정적 검증:
   `pnpm build` 통과로 TypeScript 컴파일/번들 성공을 확인했다.
6. 코드 리뷰:
   Claude 코드 리뷰를 통과했고 별도 교정 요청은 없었다.
