## 에이전트에게 위임한 작업의 범위

- `DamagePopup`, `JudgeFeedback` 컴포넌트 구현
- BattlePage에 판정/피해/회복 이펙트 연결
- 오답 시 화면 흔들림, 공격 성공/체력 회복 배너 표시
- timeout/rAF 정리(cleanup) 로직 반영

## 위임 판단 근거

애니메이션은 상태 타이밍과 cleanup이 핵심이라 누수/중복 표시 리스크가 높다.
에이전트로 구현 후 타이밍 검증을 반복하는 방식이 적합했다.

## 에이전트 산출물 검증 방법

1. 이벤트별 이펙트 검증
- 정답: 공격 배너/적 피해 팝업
- 따라치기 정답: 회복 배너/회복 팝업
- 오답/타임아웃: 피드백 배너/흔들림
- 적 자동 공격: 플레이어 피해 팝업

2. 정리 로직 검증
- 페이지 이탈/재진입 시 timeout 누수 없는지 확인

3. 정적 검증
- `pnpm build` 통과

## 에이전트가 잘못된 방향을 제시했을 때 교정 방법

- 없음. 이후 디자인 피드백에 맞춰 표시 위치/문구를 단계적으로 보정했다.
