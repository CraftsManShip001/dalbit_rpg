# T04 — 게임 상태 스토어

## 사용한 에이전트 도구

- [x] Codex
- [x] Claude
- [ ] antigravity
- [ ] 직접

## 에이전트에게 위임한 작업의 범위

- Codex에 `src/store/gameStore.ts` 전체 구현을 위임했다.
- `QuestionType`, `TurnPhase`, `GameState` 타입 정의를 포함했다.
- HP 보정 로직(`damagePlayer`, `damageEnemy`, `healPlayer`)과 스테이지/상태 전환 액션(`nextStage`, `resetGame`) 구현까지 맡겼다.
- `gameover`/`stageclear` 전환 로직은 스토어에 넣지 않고 `useGameEngine` 담당으로 분리하는 규칙을 반영하도록 했다.

## 위임 판단 근거

- T04는 Zustand 상태 모델링과 수치 보정 규칙이 중심인 태스크라 구현 패턴이 명확했다.
- 핵심 리스크가 "HP 경계값 처리 누락", "초기화 누락", "스토어 책임 범위 침범"에 집중되어 있어, Codex로 빠르게 구현하고 Claude 리뷰로 체크리스트를 검증하는 방식이 효율적이었다.

## 에이전트 산출물 검증 방법

1. 코드 기준 검증:
   - `damagePlayer`, `damageEnemy`에 `Math.max(0, ...)` 적용 확인
   - `healPlayer`에 `Math.min(playerMaxHp, ...)` 적용 확인
   - `nextStage()`에서 `stage + 1`, `enemyHp/enemyMaxHp = 150`, `turnPhase = 'idle'`, `currentQuestionType = null` 설정 확인
   - `resetGame()`에서 초기값 완전 복원 확인
   - `gameover`/`stageclear` 전환 로직이 스토어에 없는지 확인
2. 정적 검증:
   - `pnpm -s tsc --noEmit`
   - `pnpm -s eslint src/store/gameStore.ts`
   - `pnpm -s build`
3. 리뷰 검증:
   - Claude 코드 리뷰 체크리스트 전 항목 통과 확인

## 에이전트가 잘못된 방향을 제시했을 때 교정 방법

- 없음.
- 이번 T04에서는 Codex 산출물이 요구사항과 일치했고, Claude 리뷰에서도 별도 교정 요청이 발생하지 않았다.
