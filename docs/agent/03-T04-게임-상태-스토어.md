## 에이전트에게 위임한 작업의 범위

- `src/store/gameStore.ts`의 Zustand 상태 모델 구현
- 공통 전투 상태(HP, 적 이름, 층, 턴 상태)와 화면 상태(`home | battle | result`) 분리
- 액션 구현: `damagePlayer`, `damageEnemy`, `healPlayer`, `startGame`, `goHome`, `advanceFloor`, `endRun`, `resetGame`
- 최고기록(`bestFloor`)을 localStorage와 연동하도록 floor 시스템과 정합화

## 위임 판단 근거

스토어는 전체 전투 흐름의 기준점이라 타입 안정성과 상태 전이 일관성이 가장 중요했다.
에이전트로 초기 구조를 빠르게 만든 뒤, 상태 전이 규칙(`screen`, `turnPhase`)을 리뷰로 보정하는 방식이 효율적이었다.

## 에이전트 산출물 검증 방법

1. HP 경계값 검증
- `damage*`는 0 미만 방지
- `healPlayer`는 `playerMaxHp` 초과 방지

2. 상태 전이 검증
- 시작: `screen='battle'`, `turnPhase='idle'`
- 런 종료: `screen='result'`
- 홈 복귀: `screen='home'`

3. 최고기록 검증
- 층 상승/런 종료 시 `bestFloor` 갱신 및 저장

4. 정적 검증
- `pnpm build` 통과

## 에이전트가 잘못된 방향을 제시했을 때 교정 방법

- 과거 스테이지 중심 모델(`stageclear`/고정 스테이지 전환)에서 엔드리스 층 모델로 변경하며
  스토어 필드를 `stage` 중심에서 `floor + screen + bestFloor` 중심으로 재정렬했다.
