## 에이전트에게 위임한 작업의 범위

Codex에 `src/lib/yaksok.ts` 구현을 위임했다.  
구체적으로 `runYaksokCode(code: string): Promise<string[]>` 함수 작성, `YaksokSession` 생성, `stdout` 기반 출력 캡처, `runModule('main')` 실행 흐름까지 맡겼다.

Claude에는 구현 결과 코드 리뷰를 요청해 런타임 에러 전파 안정성(`stderr` 처리 방식)을 점검하도록 했다.

## 위임 판단 근거

T02는 런타임 연동의 기본 패턴이 명확하고 반복 가능한 보일러플레이트 성격이 커서 Codex 위임 효율이 높았다.  
다만 에러 전파는 런타임 내부 동작(콜백 처리 방식)에 따라 미묘하게 달라질 수 있어, 리뷰 에이전트(Claude)로 실패 시나리오를 추가 점검하는 것이 리스크를 줄인다고 판단했다.

## 에이전트 산출물 검증 방법

1. 정적 검증: `pnpm build` 실행으로 TypeScript 컴파일 및 번들 빌드 통과 확인
2. 기능 검증(출력 캡처):
`runYaksokCode('"안녕" 보여주기')` → `["안녕"]`
`runYaksokCode('1 + 2 보여주기')` → `["3"]`
`runYaksokCode('나이 = 25\n나이 보여주기')` → `["25"]`
3. 오류 검증: 존재하지 않는 변수 실행 시 `Error` throw 되는지 확인

## 에이전트가 잘못된 방향을 제시했을 때 교정 방법

초기 구현은 `stderr` 콜백 내부에서 즉시 `throw`하는 방식이었다.  
Claude 리뷰에서 "런타임이 콜백을 내부적으로 try/catch로 감싸면 throw가 상위로 전파되지 않을 수 있다"는 리스크가 제기되었다.

그래서 아래 패턴으로 즉시 교정했다.

- `let runtimeError: Error | null = null` 선언
- `stderr`에서는 `runtimeError = new Error(message)`로 수집
- `await session.runModule('main')` 완료 후 `if (runtimeError) throw runtimeError`

이 교정 후 다시 `pnpm build`를 통과시켜 최종 반영했다.
