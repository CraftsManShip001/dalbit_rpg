# ADR 01 — 코드 에디터 선택

## 상태
결정됨

## 컨텍스트

달빛약속은 한글 키워드를 사용하는 언어다. 플레이어가 `만약`, `반복`, `보여주기` 같은 한글을 에디터에 입력해야 한다. 한글 IME 입력이 에디터의 핵심 요구사항이었다.

추가로 달빛약속의 들여쓰기 규칙(4칸 스페이스)을 강제해야 했고, 판정 중에는 에디터를 잠가야 했다.

## 선택지

**Monaco Editor**
VS Code에서 사용하는 에디터. 기능이 풍부하고 생태계가 크다. 번들 사이즈가 2MB 이상으로 매우 크다. 달빛약속 전용 language mode가 없으므로 syntax highlighting 같은 고급 기능을 활용할 수 없다. 번들 사이즈 부담 대비 얻는 이점이 없다.

**CodeMirror 6**
모듈형 구조로 필요한 패키지만 설치할 수 있어 번들 사이즈가 작다. 한글 IME 입력이 안정적이다. 커스텀 키바인딩 추가가 용이하다. `Compartment`를 통해 런타임에 설정을 교체할 수 있어 `disabled` 상태 전환 구현이 깔끔하다.

**단순 textarea**
구현이 가장 단순하다. 탭 키 동작 커스터마이징이 브라우저마다 다르게 동작하고, `disabled` 상태에서 시각적 처리가 제한적이다. 다크 테마 적용도 CSS만으로는 한계가 있다.

## 결정

**CodeMirror 6**을 선택한다.

## 근거

달빛약속은 한글 입력이 핵심이다. IME 조합 중 깨짐 없이 `가나다` 입력이 가능해야 한다. CodeMirror 6은 이를 안정적으로 지원한다.

탭 키를 4칸 스페이스로 강제하는 커스텀 키바인딩이 필요했다. CodeMirror 6의 `keymap` 확장으로 간단히 구현했다.

Monaco는 달빛약속 전용 language mode가 없는 상황에서 plain text mode로만 쓰기에는 번들 사이즈 부담이 지나치게 크다.

textarea는 `disabled` 상태 처리와 들여쓰기 강제에 한계가 있어 RPG 테마의 시각적 잠금 표현이 어렵다.

## 결과

한글 IME 입력 브라우저 테스트에서 `가나다` 조합 중 깨짐 없음을 확인했다. 탭 키 입력 시 4칸 스페이스 삽입도 검증했다. `Compartment.reconfigure()`로 `disabled` 상태 전환이 즉시 반영된다.
