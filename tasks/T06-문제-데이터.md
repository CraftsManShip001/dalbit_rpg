### `src/data/questions.ts`

타입 정의:

```ts
export interface QuickQuestion {
  type: 'quick'
  prompt: string
  timeLimit: number
  expectedOutputs: string[]
  hint?: string
}

export interface CopyQuestion {
  type: 'copy'
  prompt: string
  codeToType: string
  timeLimit: number
}

export interface AlgorithmQuestion {
  type: 'algorithm'
  prompt: string
  timeLimit: number
  expectedOutputs: string[]
  hpPenalty: number
}

export type Question = QuickQuestion | CopyQuestion | AlgorithmQuestion

export interface StageQuestions {
  stage: number
  questions: Question[]
}
```

문제 데이터 (스테이지 1 — 각 유형 최소 3개):

**빠른 문제 (quick):**
```
"안녕하세요" 보여주기 → expectedOutputs: ["안녕하세요"]
나이 = 25 \n 나이 보여주기 → ["25"]
10 + 37 보여주기 → ["47"]
"달빛" + "약속" 보여주기 → ["달빛약속"]
```

**따라치기 (copy):**
```
3번 반복\n    "공격!" 보여주기
만약 1 > 0 이면\n    "참" 보여주기\n아니면\n    "거짓" 보여주기
이름 = "용사"\n이름 보여주기
```

**알고리즘 (algorithm):**
```
1부터 10까지 합 반복문으로 출력 → ["55"]
[3, 1, 4, 1, 5]에서 가장 큰 수 출력 → ["5"]
구구단 3단 출력 (3, 6, 9, ... 27) → ["3", "6", ..., "27"]
```

스테이지 2는 연산/문자 이어붙이기 포함한 문제로 각 유형 3개.

## 완료 기준

- [ ] `STAGE_CONFIGS` 1, 2 스테이지 정의됨
- [ ] `questionWeights` 합계 == 1.0
- [ ] 스테이지 1 빠른문제 4개, 따라치기 3개, 알고리즘 3개 이상
- [ ] 스테이지 2 각 유형 3개 이상
- [ ] 따라치기 `codeToType` 코드의 들여쓰기가 정확히 4칸
- [ ] 알고리즘 `expectedOutputs`가 실제 달빛약속 실행 결과와 일치 (직접 검증 필요)
- [ ] `any` 타입 없음

## Claude 코드 리뷰 체크리스트

- `Question` discriminated union 타입 올바른가 (`type` 필드로 분기)
- `questionWeights` 합이 1.0인가
- 따라치기 `codeToType` 들여쓰기 4칸인가 (탭 아님)
- 알고리즘 `expectedOutputs` 값이 실제 실행 결과와 맞는가

## ⚠️ 정현이 직접 검증 필요

알고리즘 문제의 `expectedOutputs`는 달빛약속 온라인 데모(https://dalbit-yaksok.postica.app)에서 직접 실행해서 실제 출력값 확인 후 채울 것. Codex가 추정으로 넣으면 판정 로직 전체가 오동작함.
